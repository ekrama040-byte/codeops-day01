# Day 1 Reflection

### 1. What is the difference between Git and GitHub?
Git is a local command-line tool that tracks changes to your files on your computer. GitHub is an online cloud platform where you host your Git repositories to save backups and collaborate with others.

### 2. What is the purpose of the terminal?
The terminal is a text-based interface used to interact with your computer's operating system directly by typing commands instead of clicking buttons.

### 3. What does git add do?
It moves your modified or new files to the "staging area," telling Git that you want to include these specific changes in your next snapshot.

### 4. What is the purpose of git commit?
It saves a permanent snapshot of your staged changes into your project's Git history, acting like a checkpoint.

### 5. Write the daily Git workflow in the correct order.
1. Modify or create files in your workspace.
2. git status (to view changes).
3. git add <file> (to stage changes).
4. git commit -m "message" (to save changes locally).
5. git push origin main (to upload changes to GitHub).
